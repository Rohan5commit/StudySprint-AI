# Final Round Build Plan

## 8-hour feature priority
1. Landing page + polished product story
2. Onboarding form for subject, exam date, weak chapters, study hours, learning style
3. Generate one complete study pack
4. Dashboard with summary, checklist, readiness, and weak topics
5. Quiz mode
6. Flashcard mode
7. Study plan / timetable view
8. Last-minute revision mode panel
9. Nice-to-have polish only after the core demo is stable

## What to build first
- landing page
- seeded sample data
- onboarding form
- one working study-pack generator
- dashboard shell

## What to fake safely if time is short
- multi-subject support can be framed as future work; demo one subject flow well
- plan updates can be client-side rules instead of advanced AI reasoning
- progress sync can stay browser-local instead of using auth or databases
- last-minute mode can be a smart compressed view of existing outputs

## Final polished demo must include
- one-click instant demo
- visible exam date + study hours input
- summary + checklist
- quiz + flashcards
- personalized timetable
- last-minute revision mode

## Cut list if time runs low
1. file upload enhancements
2. advanced animations
3. account system
4. multi-user collaboration
